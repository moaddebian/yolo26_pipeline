Architecture Double Tête
YOLO26 conserve l'architecture duale de YOLOv10 : un backbone et un neck partagés alimentent deux têtes parallèles — O2M (one-to-many) et O2O (one-to-one). Les caractéristiques sont détachées (stop-gradient) avant la tête O2O.

ProgLoss — Pondération Progressive
ProgLoss est un ordonnanceur dynamique qui fait décroître le poids de la tête O2M et croître celui de la tête O2O au fil des époques. En début d'entraînement, le signal O2M (riche, multi-ancres) domine ; en fin, le signal O2O (une ancre par objet) prend le relais.
Évolution des poids w_o2m et w_o2o sur 500 époques
w_o2m (décroissant)
w_o2o (croissant)
w_o2m(t) = max(1 − t/(epochs−1), 0) × (0.8 − 0.1) + 0.1
w_o2o(t) = 1.0 − w_o2m(t)

L_total(t) = w_o2m(t) × L_o2m + w_o2o(t) × L_o2o

// Époque 0 : w_o2m=0.80, w_o2o=0.20 — O2M domine (apprentissage supervisé riche)
// Époque 250 : w_o2m=0.45, w_o2o=0.55 — Transition progressive
// Époque 499 : w_o2m=0.10, w_o2o=0.90 — O2O domine (préparation à l'inférence)

TaskAlignedAssigner (TAL)
Le TAL utilise une métrique d'alignement combinant le score de classification et l'IoU pour sélectionner les ancres positives. Le mode O2M sélectionne les top-10 ancres ; le mode O2O sélectionne d'abord top-7 puis réduit à topk2=1 (la meilleure).
Métrique d'alignement
M = scoreα × IoUβ

avec α = 0.5 et β = 6.0

// β=6.0 ≫ α=0.5 : la localisation est fortement prioritaire sur la classification
// Cohérence O2O/O2M : mêmes α, β pour les deux têtes (Consistent Matching)

O2M — One-to-Many

TAL(topk=10)
Signal superviseur riche : 10 ancres reçoivent un gradient positif par objet, accélérant la convergence du backbone et du neck partagés.

O2O — One-to-One

TAL(topk=7, topk2=1)
Sélection unique : Parmi 7 candidates pré-sélectionnées, seule celle avec le M le plus élevé est retenue (topk2=1). Pas de doublons → pas besoin de NMS.

STAL — Small-Target-Aware Label Assignment
STAL détecte les petits objets (<8px) dont la boîte englobante est trop petite pour être assignée à l'échelle P3 (stride=8). Il agrandit virtuellement la boîte au stride suivant (P4, stride=16) pour garantir un pool d'ancres candidates suffisant.
Mécanisme d'agrandissement STAL
AVANT STAL
APRÈS STAL
Règle STAL :

SI max(bbox_w, bbox_h) < stride[0]  // objet plus petit que 8px
ALORS agrandir la bbox au stride[1] = 16
       → sélection d'ancres sur la feature map P4 (stride=16)

// Garantit ≥1 ancre candidate même pour les objets de <8px
// Critique pour piétons éloignés en images fisheye drone

Graphe de Dépendances des Composants
Chaque innovation de YOLO26 dépend des autres. Ce diagramme montre pourquoi chaque composant est nécessaire et ce qui se passe sans lui.
Chaîne de dépendances
Composant Pourquoi nécessaire Sans ce composant…
Double Tête
Fondation
Permet d'avoir un signal O2M riche à l'entraînement (convergence rapide du backbone) tout en produisant une sortie O2O sans doublons à l'inférence.
→ Avec O2O seul : convergence lente, signal superviseur trop faible. Avec O2M seul : NMS obligatoire à l'inférence.
ProgLoss
Ordonnanceur
Gère la transition douce d'un signal dominé par O2M vers un signal dominé par O2O. Évite les conflits de gradient entre les deux têtes.
→ Avec poids fixes (0.5/0.5) : les deux têtes se disputent le backbone, instabilité de convergence. −0.3% à −0.5% mAP.
TAL O2O
Assignation
Sélectionne exactement 1 ancre par objet (topk2=1) avec la meilleure métrique d'alignement M. Élimine structurellement les doublons.
→ Avec topk2>1 : plusieurs prédictions par objet → doublons → NMS nécessaire à nouveau.
TAL O2M
Superviseur
Fournit 10 ancres positives par objet : gradient dense et signal superviseur riche pour le backbone et le neck partagés.
→ Sans O2M : le backbone reçoit un signal trop épars (1 ancre/objet), convergence lente, features de faible qualité.
STAL
Petits objets
Garantit ≥1 ancre candidate pour les objets <8px en les promouvant au stride supérieur. Critique pour piétons éloignés en vue drone.
→ Sans STAL : les objets <8px n'ont aucune ancre candidate → perdus à l'entraînement → rappel effondré sur petits objets.
detach()
Isolation
Empêche les gradients de la tête O2O de modifier les features du backbone. Seule la tête O2M (signal riche) entraîne le backbone.
→ Sans detach : la tête O2O (1 ancre/objet) corrompt les features partagées avec un gradient trop bruyant.
Sans DFL
Simplification
Élimine l'opération softmax (DFL) du graphe, permettant un export natif sur NPU/DSP/microcontrôleurs sans opérations dynamiques.
→ Avec DFL : export difficile (softmax non supporté sur certains NPU), latence accrue sur hardware embarqué.
Sources : Ultralytics [11] · Wang et al. [10] · Chakrabarty [13]
Flux d'Appels du Code
Séquence d'appels de fonctions pendant une passe forward d'entraînement, depuis l'entrée de l'image jusqu'au calcul de la perte totale pondérée par ProgLoss.
Séquence d'appels — Entraînement
forward(images)
Point d'entrée. L'image passe par le backbone et le neck.
def forward(self, images):
  features = self.backbone(images)  # C2f / RepVGG
  fpn_out = self.neck(features)    # P3, P4, P5
  return self.forward_head(fpn_out)
forward_head(fpn_out)
Les features FPN alimentent les deux têtes. Les features sont détachées avant la tête O2O.
def forward_head(self, fpn_out):
  out_o2m = self.head_o2m(fpn_out)           # gradient normal
  out_o2o = self.head_o2o(fpn_out.detach())  # stop-gradient
  return out_o2m, out_o2o
E2ELoss(out_o2m, out_o2o, targets)
Calcule les deux pertes (O2M et O2O) et les combine avec ProgLoss.
def E2ELoss(self, out_o2m, out_o2o, targets, epoch):
  loss_o2m = self.compute_loss(out_o2m, targets, mode="o2m")
  loss_o2o = self.compute_loss(out_o2o, targets, mode="o2o")
  return self.prog_loss(loss_o2m, loss_o2o, epoch)
TAL(topk=10) — Assignation O2M
Le TaskAlignedAssigner sélectionne les 10 meilleures ancres par objet selon M = score0.5 × IoU6.0.
# Branche O2M
assigner = TaskAlignedAssigner(topk=10, alpha=0.5, beta=6.0)
matches = assigner(pred_scores, pred_bboxes, gt_bboxes)
TAL(topk=7, topk2=1) — Assignation O2O
Pré-sélectionne 7 candidates, puis retient uniquement la meilleure (topk2=1) par objet.
# Branche O2O

matches = assigner(pred_scores, pred_bboxes, gt_bboxes)
BboxLoss (CIoU) + BCE (classification)
Pour chaque branche : perte de localisation CIoU sur les boîtes + BCE (Binary Cross-Entropy) sur les scores.
loss_bbox = CIoULoss(pred_bboxes[pos], gt_bboxes[pos])
loss_cls = BCEWithLogitsLoss(pred_scores, target_scores)
branch_loss = loss_bbox + loss_cls
ProgLoss(loss_o2m, loss_o2o, epoch)
Pondération dynamique : w_o2m décroît de 0.8→0.1, w_o2o croît de 0.2→0.9.
def prog_loss(self, l_o2m, l_o2o, epoch):
  w = max(1 - epoch / (self.epochs - 1), 0)
  w_o2m = w * (0.8 - 0.1) + 0.1
  w_o2o = 1.0 - w_o2m
  return w_o2m * l_o2m + w_o2o * l_o2o
